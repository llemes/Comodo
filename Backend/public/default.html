API routes

All routes listed below follow the same pattern:

/route

  • GET: gets all entries
  • POST: creates an entry

/route/id

  • GET: gets entry with appropriate id
  • PUT: updates entry with appropriate id
  • DELETE: deletes entry with appropriate id

/users

Sample object for POST/PUT:

{ "fullName": "John Doe", "email": "john.doe@example.com", "username": "johndoe", "password": "mypassword", "role": "member", "organisationId": "existingOrganisationId" }

Note: full name is required. E-mail is required and must be unique. Username is required and must be unique. Password is required.

/resources

Sample object for POST/PUT:

{ "name": "Some name", "occupied": false, "resourceType": "existingResourceTypeId", "organisationId": "existingOrganisationId" }

Note: occupied is a boolean value

/resourcetypes

Sample object for POST/PUT:

{ "name": "Required Unique Name", "imageUrl": "https://imgur.com/example" }

Note: name must be unique

/organisations

Sample object for POST/PUT:

{ "name": "Organisation Name" }

Note: name is required and must be unique.
